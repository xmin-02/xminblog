## 들어가며

운영 중인 서버에 `df -h` 쳐봤다가 디스크가 99%까지 차 있어서 당황한 경험, 다들 한 번쯤 있으실 겁니다. 로그 파일, APT 캐시, `/tmp` 쓰레기들이 쌓여서 시간이 지나면 몇 GB씩 먹히죠.

매번 수동으로 정리하기 귀찮아서 스크립트 하나를 만들어뒀습니다. 한 줄로 실행 가능합니다.

## 설치 및 실행

터미널에 아래 명령어 한 줄만 입력하세요.

```bash
curl -fsSL https://xmin.blog/scripts/clean.sh | sudo bash
```

실행하면 다음과 같은 출력이 나옵니다.

```
[+] 불필요한 임시 파일 검색 중...
[+] /tmp 디렉토리 캐시 정리...
[+] APT 캐시 비우기...
[+] 로그 파일 압축 및 삭제...
[✓] 정리 완료! 확보된 공간: 4.2GB
```

## 스크립트가 하는 일

- `/tmp` 및 `/var/tmp` 의 7일 이상 된 파일 제거
- `apt clean`, `apt autoremove` 실행
- `/var/log` 하위의 오래된 로그 압축 및 삭제
- `~/.cache` 정리

Ubuntu 20.04 / 22.04 / 24.04 에서 테스트 완료되었습니다. 다른 배포판(Debian, CentOS)에서도 APT 부분만 빼면 문제없이 동작할 겁니다.

## 주의사항

루트 권한이 필요합니다. `sudo` 빼먹으면 일부 명령이 실패합니다.

그리고 — 당연한 얘기지만 **모든 자동화 스크립트는 내용을 확인하고 실행**하는 것이 좋습니다. 이 블로그 스크립트도 마찬가지로요.
